  <action>
    **Create `src/utils/vaultService.js`** — the vault metadata persistence layer. Follow the AsyncStorage pattern already established in `src/utils/storage.js` (keys with `kwestup_` prefix, version suffix).

    Implement these exports:
    - `VAULTS_KEY = 'kwestup_vaults_v5.0'` — AsyncStorage key for vault configs array
    - `ACTIVE_KEY = 'kwestup_activeVault_v5.0'` — AsyncStorage key for active vault ID
    - `migrateToVaultSystem()` — One-time migration from old `Notes/` structure to `Notes/Vaults/default/`. 
      1. Check if `Notes/Vaults/` exists — if so, skip (already migrated)
      2. Create `Notes/Vaults/` and `Notes/Vaults/default/` directories
      3. Read all items from `Notes/`, skip `Vaults/` directory, move everything into `Notes/Vaults/default/`
      4. Create default VaultConfig `{ id: 'default', name: 'My Vault', path: Notes/Vaults/default/, createdAt, updatedAt }`
      5. Save vaults array to AsyncStorage, set activeVaultId to 'default'
      6. Use `FileSystem.moveAsync()` for files and folders; if move fails, fall back to copy+delete
    - `getVaults()` — Read and parse VAULTS_KEY from AsyncStorage, return VaultConfig[] (default [])
    - `saveVaults(vaults)` — Stringify and write to AsyncStorage
    - `createVault(name)` — Generate ID via `Date.now().toString()`, create directory, save vault
    - `deleteVault(vaultId)` — Delete vault directory, remove from AsyncStorage; if deleted vault was active, switch to first remaining vault
    - `renameVault(vaultId, newName)` — Update name + updatedAt in vaults array
    - `getActiveVaultId()` — Read ACTIVE_KEY from AsyncStorage
    - `setActiveVaultId(id)` — Write ACTIVE_KEY to AsyncStorage
    - `getVaultPath(vaultId)` — Return `${FileSystem.documentDirectory}Notes/Vaults/${vaultId}/`
    - `ensureVaultsDir()` — Create `Notes/Vaults/` with intermediates if it doesn't exist (used by migration)

    **Create `src/utils/vaultImport.js`** — the vault import function:
    - `importMDFilesAsVault(vaultName)`:
      1. Call `DocumentPicker.getDocumentAsync({ type: ['text/markdown', 'text/plain'], multiple: true, copyToCacheDirectory: true })`
      2. If cancelled, return null
      3. Create vault via `createVault(vaultName || 'Imported')`
      4. For each picked file: read content with `FileSystem.readAsStringAsync()`, write to vault path preserving original filename
      5. Return the created vault config

    **Install expo-document-picker:**
    ```bash
    npx expo install expo-document-picker
    ```
    Verify the import works with `import * as DocumentPicker from 'expo-document-picker'`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AsyncStorage ↔ App State | Vault metadata read from AsyncStorage could be null, malformed, or stale |
| Filesystem ↔ vaultService | Filesystem operations may fail if paths don't exist or permissions are denied |
| expo-document-picker ↔ vaultImport | Picked file URIs may not be readable by expo-file-system on all Android devices |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | vaultService.createVault | accept | Vault IDs are Date.now() strings — collision risk is negligible for local usage |
| T-08-02 | Tampering | fileStorage.js vault path | mitigate | All file operations use `getVaultPath(vaultId)` which constructs paths from the vault ID, not user input. Folder/title names are sanitized with replace() to remove path separators. |
| T-08-03 | Tampering | vaultImport.js file copies | mitigate | Source URIs from document picker are read-only; destination is always within the vault's sandboxed directory. Path traversal not possible since `copyToCacheDirectory: true` provides safe URIs. |
| T-08-04 | Denial of Service | AsyncStorage quota | accept | Vault metadata is <1KB per vault. AsyncStorage ~6MB limit on Android won't be hit by vault configs alone. Note content stays on filesystem. |
| T-08-05 | Information Disclosure | vaultImport.js | accept | Only .md files are imported. No PII or sensitive data beyond what user explicitly selects. |
</threat_model>
