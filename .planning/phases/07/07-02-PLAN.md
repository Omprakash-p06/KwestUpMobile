**Key architectural note:** The app uses a monolithic state-in-App.js pattern. Screens receive `currentTheme` as a prop. This plan does NOT change the state management or prop-drilling architecture — only the visual rendering inside each screen.

**Theme system:** `src/theme/colors.js` exports `themes[selectedThemeName][themeMode]` providing colors. The glass effect uses translucent overlays on top of whatever background color the theme provides. The LiquidGlassBackground adds animated blobs behind everything. Glass cards sit on top of the blob layer.

**Components available after Plan 07-01:**
- `LiquidGlassCard` - Skia glass card (BackdropFilter + Blur + ColorMatrix + sheen + rim border)
- `LiquidGlassBackground` - Animated fluid blob background
- glass prop on `CustomButton`, `CustomTextInput`, `CustomBadge`

**Component contracts (from 07-01):**

LiquidGlassCard:
```jsx
<LiquidGlassCard
  style={...}       // width/height/margin/etc
  blurRadius={20}   // optional, 15-25
  glassTint="rgba(255,255,255,0.12)"  // optional
  borderColor="rgba(255,255,255,0.28)" // optional
  borderWidth={1.5}  // optional
  borderRadius={24}  // optional
  theme={currentTheme}
>
  {children}
</LiquidGlassCard>
```

LiquidGlassBackground:
```jsx
<LiquidGlassBackground
  blobColors={['#FF007F', '#6A0DAD']}  // optional, auto-selects based on themeMode
  backgroundColor={currentTheme.background}
  intensity={0.35}
>
  <ScrollView>...screens content...</ScrollView>
</LiquidGlassBackground>
```

glassTheme.js tokens:
```javascript
LIQUID_MATRIX, BLUR_RADIUS_DEFAULT, SHEEN_TINT, RIM_BORDER_COLOR, RIM_BORDER_WIDTH, GLASS_BORDER_RADIUS, BLOB_COLORS_DARK, BLOB_COLORS_LIGHT
```

# Pattern: Replacing `<CustomCard>` with `<LiquidGlassCard>`

**Before:**
```jsx
<CustomCard style={{ backgroundColor: currentTheme.cardBackground }}>
  <Text>Content</Text>
</CustomCard>
```

**After:**
```jsx
<LiquidGlassCard style={{ /* width/height/margin - no backgroundColor needed */ }} theme={currentTheme}>
  <Text>Content</Text>
</LiquidGlassCard>
```

# Pattern: Wrapping screens with glass background

**Before:**
```jsx
<ScrollView style={styles.tabContentScroll}>
  <View style={styles.tabContent}>
    ...screens content...
  </View>
</ScrollView>
```

**After:**
```jsx
<LiquidGlassBackground backgroundColor={currentTheme.background} themeMode={themeMode}>
  <ScrollView style={styles.tabContentScroll}>
    <View style={styles.tabContent}>
      ...screens content...
    </View>
  </ScrollView>
</LiquidGlassBackground>
```

# Pattern: Glass-styled drawer items
- Drawer background: translucent dark overlay on blob background
- Active item: semi-transparent primary tint with rim border
- Text colors: light/white for contrast on dark glass

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04 | I (Information Disclosure) | LiquidGlassBackground blobs | accept | Blobs are purely decorative gradient shapes with no content. Background gradient colors use theme-provided values. No user data rendered through blob layer. |
| T-07-05 | I (Information Disclosure) | BackdropFilter in cards | accept | BackdropFilter captures pixels behind glass cards and applies heavy blur (radius 20). Blur + sheen tint obfuscates underlying content. Content behind cards is already part of the same app UI. |
| T-07-06 | S (Spoofing) | Glass-styled dialog modals | accept | Modals retain their existing backdrop press handling and dismiss logic. Glass styling does not change interaction model. User intent is preserved through existing confirmation patterns. |
</threat_model>
