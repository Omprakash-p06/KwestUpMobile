**Current UI architecture:** All state lives in `App.js`. Screens in `src/screens/` are passed `currentTheme` object from `src/theme/colors.js`. Components in `src/components/` accept `theme` prop and read `currentTheme.cardBackground`, `currentTheme.primary`, `currentTheme.text`, etc. for styling. The existing `CustomCard` (`src/components/CustomCard.js`) is a simple `<View>` with backgroundColor from theme.

**Key philosophy per research:** CPU-based blurs on Android Views cause frame drops. All blurs must execute in Skia `<BackdropFilter>` canvas blocks. Keep Blur radius between 15-25. Always clip BackdropFilter to card bounds. Never use unbounded backdrop blurs.
