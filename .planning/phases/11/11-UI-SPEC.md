## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Compact element spacing, card padding tight |

## Color

All colors are dynamic via `currentTheme` from `src/theme/colors.js`. The five accent themes (dribbble, clean, blue, green, purple) each have light/dark/amoled variants. The following contract binds where each semantic role applies within this phase.

| Role | Value (dark default) | Usage |
|------|---------------------|-------|
| Dominant (60%) | `currentTheme.background` (#131313 dark / #E4E2E1 light / #000000 amoled) | Page background, scroll area |
| Secondary (30%) | `currentTheme.cardBackground` (#20201F dark / #FFFFFF light / #0E0E0E amoled) | DATA ARCHIVE card surface, modal dialog background |
| Accent (10%) | `currentTheme.primary` (varies: #FFFFFF dribbble dark / #64B5F6 blue dark / etc.) | CTA buttons (EXPORT ENCRYPTED, RESTORE BACKUP), progress bar fill, passphrase field focus border, modal action buttons, icon highlights |
| Destructive | `currentTheme.error` (#FF5555 dark / #FF0000 light) | Cancel/CONTINUE warning? — No, destructive warning uses error color for the warning text/border only; CONTINUE is accent, CANCEL is outline |
| Text Primary | `currentTheme.text` | All headings, labels, button text |
| Text Secondary | `currentTheme.secondaryText` | Descriptions, status text, placeholders |
| Border | `currentTheme.border` | Card borders, dividers, progress bar track |

Accent reserved for: CTA buttons (EXPORT ENCRYPTED, RESTORE BACKUP), progress bar fill indicator, passphrase modal confirm action, focus/highlight on active passphrase field.

Destructive reserved for: Import warning border/accent ("This will overwrite all existing data"), dismiss/close icon in modal header.

**Implicit from existing `src/theme/colors.js`** — no new tokens introduced.

## Interaction States

### DATA ARCHIVE Card (in Settings)

| State | Visual | Behavior |
|-------|--------|----------|
| **Default** | LiquidGlassCard with chassis header "DATA ARCHIVE.sys", description explaining .kwestup backup, two stacked buttons: EXPORT ENCRYPTED | RESTORE BACKUP | — |
| **Processing (Export)** | Inline progress bar renders below description, replacing the EXPORT button area (not overlay). Progress bar shows percentage (matching AI download pattern in SettingsScreen). Export button disabled. | `CryptoJS.AES.encrypt` + `expo-sharing` pipeline active |
| **Processing (Import)** | Same inline progress bar area active. Import button disabled. | `expo-document-picker` → decrypt → restore sequence active |
| **Error** | Error banner inside card (amber/error background + message) replaces progress bar. Dismissible. | Tap dismiss restores default state |

### Passphrase Modal

| State | Visual | Behavior |
|-------|--------|----------|
| **Closed** | Not rendered | — |
| **Open (Export mode)** | Modal overlay with title "EXPORT ENCRYPTED", two password fields (ENTER PASSPHRASE, CONFIRM PASSPHRASE), hint text, CONFIRM / CANCEL buttons | Both fields must match to enable CONFIRM |
| **Open (Import mode)** | Modal overlay with title "RESTORE BACKUP", single password field (ENTER PASSPHRASE), warning text about overwriting, CANCEL / CONTINUE buttons | CONTINUE enabled when field non-empty |
| **Field mismatch (Export)** | Red error border on confirm field + inline error text "PASSPHRASES DO NOT MATCH" | Confirm stays disabled |
| **Decrypt error (Import)** | Red error border + inline error text "INVALID PASSPHRASE OR CORRUPTED ARCHIVE" | Stays open for retry |
| **Confirm loading** | CONFIRM/CONTINUE button shows ActivityIndicator, fields disabled | Processing animation during encryption/decryption |

## Interaction Flows

### Flow A: Export Flow

```
1. User taps [EXPORT ENCRYPTED] button in DATA ARCHIVE card
2. DataArchiveModal opens in EXPORT mode
3. User enters passphrase in ENTER PASSPHRASE field
4. User re-enters passphrase in CONFIRM PASSPHRASE field
5. If mismatch → inline error, CONFIRM stays disabled
6. If match → CONFIRM enabled
7. User taps CONFIRM
8. Modal closes, inline progress bar appears in card with "ENCRYPTING..."
9. Data collection + AES encryption runs
10. expo-sharing opens native share sheet
11. On complete → progress bar removed, success feedback
12. On error → error banner in card
```

### Flow B: Import Flow

```
1. User taps [RESTORE BACKUP] button in DATA ARCHIVE card
2. expo-document-picker opens native file picker for .kwestup
3. After file selected → DataArchiveModal opens in IMPORT mode
4. Warning displayed: "WARNING: This will overwrite all existing data"
5. User enters passphrase in ENTER PASSPHRASE field
6. User taps CONTINUE
7. Modal closes, inline progress bar appears in card with "DECRYPTING..."
8. AES decryption runs
9. If wrong password → error inline in modal, retry
10. If correct → confirmation dialog: "Archive restored successfully. Reloading..."
11. App state reloads with restored data
12. On error → error banner in card
```
