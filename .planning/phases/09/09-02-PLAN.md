    **Step B — Add debounced widget update effect** (after the existing `saveData` useEffect at line 258-283):

    Insert a new `useEffect` that runs when timer or task state changes:
    ```javascript
    // Debounced foreground widget updates — pushes state to home-screen widgets
    // when app is in foreground. Background updates handled by widget-task-handler.
    useEffect(() => {
      if (!isInitialized) return;

      // Debounce: only update every 2 seconds to avoid excessive bridge traffic
      const debounceTimer = setTimeout(() => {
        const widgetData = {
          remaining: timerRemaining,
          isRunning: isTimerRunning,
          dailyTaskCount: dailyTasks.length,
          dailyTasksCompleted: dailyTasks.filter(t => t.completed).length,
        };

        requestWidgetUpdate({
          widgetName: 'FocusTimer',
          renderWidget: () => (
            <FocusTimerWidget
              remaining={widgetData.remaining}
              isRunning={widgetData.isRunning}
            />
          ),
        });

        requestWidgetUpdate({
          widgetName: 'DailyTasks',
          renderWidget: () => (
            <DailyTasksWidget
              dailyTaskCount={widgetData.dailyTaskCount}
              dailyTasksCompleted={widgetData.dailyTasksCompleted}
            />
          ),
        });
      }, 2000); // 2-second debounce — per RESEARCH.md Open Question 1 recommendation

      return () => clearTimeout(debounceTimer);
    }, [timerRemaining, isTimerRunning, dailyTasks, isInitialized]);
    ```

    **Placement:** Insert this useEffect between the existing `saveData` useEffect and the `showConfirmation` function (around line 283 on the line before `const showConfirmation`).

    **Why debounce to 2 seconds (from RESEARCH.md):**
    - Timer ticks every 1 second → requesting widget update every tick creates excessive bridge traffic
    - 2-second debounce balances responsiveness with bridge load
    - Widget shows "smooth" countdown (updates every 2 seconds instead of every second)
    - Users looking at the widget will see a timer that updates, just at half the frequency of the in-app timer

    **DO NOT modify or remove any existing code in App.js.** Only add:
    - 3 import statements at the top
    - 1 useEffect block in the component body (before `const showConfirmation`)
