# Plan 05-01: llama.rn Integration & GGUF Model Loader

This plan establishes the foundation for on-device AI in KwestUp Mobile using `llama.rn` and setting up the model downloader/loader.

## Goal
Integrate the offline AI engine (`llama.rn`) and build a local service that handles download, progress reporting, initialization, and unloading of the Qwen3-0.6B LLM GGUF model file.

## Proposed Changes

### Configuration & Dependency Layer

#### [MODIFY] [package.json](file:///home/omprakash/Documents/KwestUpMobile/package.json)
- Add `"llama.rn": "^0.12.4"` to dependencies.
- Add `"expo-build-properties": "~0.12.1"` (or appropriate version) for configuring native static frameworks.

#### [MODIFY] [app.json](file:///home/omprakash/Documents/KwestUpMobile/app.json)
- Add `expo-build-properties` plugin configuration:
  ```json
  [
    "expo-build-properties",
    {
      "ios": {
        "useFrameworks": "static"
      }
    }
  ]
  ```

### Utility Layer

#### [NEW] [aiService.js](file:///home/omprakash/Documents/KwestUpMobile/src/utils/aiService.js)
- Implement `isModelDownloaded()` to check cache status.
- Implement `downloadModel(onProgress)` using `expo-file-system` to download from HuggingFace to device document directory.
- Implement `loadModel()` using `initLlama()` to spin up context with 2048 token window.
- Implement `unloadModel()` using `releaseAllLlama()` to free memory.
- Implement system prompt and completions for `summarizeNote` and `extractTasksFromNote`.
